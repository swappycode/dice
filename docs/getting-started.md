# Getting Started

## Prerequisites
- Windows 11 (primary dev target), macOS/Linux should work for the backend
- [rustup](https://rustup.rs) — the pinned toolchain (see `rust-toolchain.toml`) auto-installs on first `cargo` use
- Node 22+ and npm (desktop client only)
- Docker (for Postgres; Redis/NATS only needed in the `full` profile)
- No protoc needed — it is vendored via `protoc-bin-vendored` in `crates/protocol/build.rs`
  (note: the vendored Windows binary is win32, runs via WOW64; Windows-on-ARM is unsupported)

## One-time setup
```powershell
./scripts/bootstrap.ps1        # installs just + sqlx-cli (pinned 0.8.x)
Copy-Item .env.example .env
just infra-up                  # Postgres + Redis + NATS via docker compose
just db-setup                  # create DB + run migrations
```

## Daily loop
```powershell
just dev          # monolith, dev-lite profile (in-proc bus, memory cache, docker Postgres)
just client       # desktop client (separate terminal)
just check        # the PR gate: fmt, clippy -D warnings, tests, aws-lc-sys gate
```

No database? Builds still work: the committed `.sqlx/` cache makes `cargo check` fully offline
(`SQLX_OFFLINE=true`). See [database.md](database.md).

## Gotchas on Windows
- **Never let `aws-lc-sys` into the tree** (needs NASM/CMake; the MSYS2 MinGW cmake on PATH would
  poison MSVC builds). `just gate-aws-lc` enforces this; all TLS crates pin the `ring` provider.
- Dev TLS certs and JWT keys are auto-generated by the monolith on first boot into `dev/`
  (gitignored). The client trusts the dev CA via the `DICE_DEV_CA` env var — never disable verification.
